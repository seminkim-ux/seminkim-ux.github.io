APP

노브랜드버거의 주문채널(앱, 모바일웹, PC웹) 구축 프로젝트에서 메인으로 기획을 진행하였습니다.

가입부터 주문, 주문 후 피드백, 사용자 이벤트 등 전체 영역을 설계하였습니다.

테스트를 통해 오류를 보완하고, 서비스를 런칭하였습니다.

ROLE UX PLANNING / 70% PROGRESS Increased 20% App Order

Overview

외부 채널에 의존한 온라인 주문을
브랜드 고유의 서비스로.

노브랜드버거는 배민, 요기요 등 외부 채널로만 온라인 주문을 운영하고 있었습니다.
자체 채널로 비용을 절감하고, 사용자와 관계를 구축해야 했습니다.
특히 Why pay more라는 브랜드 콘셉트와 어울리는 합리적인 주문 UX를 이루기 위해,
빠른 가입과 주문이라는 목표를 실현했습니다.

Process

UX 기획 전체 과정을 총괄하여 진행하였습니다.

Define

  • 경쟁사 분석 및 평가
  • UX 트렌드 조사
  • 서비스 트렌드 조사
  • 브랜드 분석

다양한 프랜차이즈 앱을 휴리스틱 사용성
관점에서 분석하였습니다.
UX와 서비스 트렌드를 조사하고,
브랜드 특성을 연구하였습니다.

Ideation

  • 코크리에이션 워크샵
  • Service Concept
  • 특색있는 Key 기능 연구

마케팅 부서, 개발팀, IT담과 정기적인
워크샵을 통해 니즈를 분석하고
Concept를 수립하였습니다.
서비스만의 특색있는 Key 기능을
정의하였습니다.

Prototype

  • Information Architecture
  • Wireframe 작성
  • 알림톡 메시지 작성

전체 구조를 설계하고,
메인, 가입, 주문, 결제 등 각 세부화면의
와이어프레임을 작성하였습니다.
피드백 메시지로 발송되는 상황별 알림톡
메시지를 작성하였습니다.

Test

  • 테스트 시나리오 작성
  • 임직원 대상 테스트 진행
  • 알림톡 테스트
  • 필드테스트

테스트 시나리오를 작성하였습니다.
임직원 대상 테스트 시 진행을 맡아
클라이언트와 개발자 사이를 조율하였습니다.
알림톡 발송 규칙에 맞는지 확인하는 테스트와,
매장 내 주문 테스트도 진행하였습니다.

Client Interview

사용성에 공감할 수 있는 서비스를 만든 후,
더욱 확장된 플랫폼으로 발전해야 합니다.

HDC와 HDC랩스 임직원을 대상으로 인터뷰를 진행하였습니다.
기능만 제공했던 기존의 서비스에서, 사용자 관점의 UX를 제공하는 목표를 가지고 있었습니다.
다양한 기능을 통합하여 잘 제공한 후, 추후에는 아이파크몰과 연계한 세일즈까지 염두에 두었습니다.

User Research

사용자들은 제어 기능에 익숙하지 않고,
앱 사용 필요성을 느끼지 못했습니다.
