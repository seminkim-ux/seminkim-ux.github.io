Overview

개별 앱으로 분산된 IoT 기능을 통합.
IPARK만의 특색을 살린 UX 제시.

개편 전 아이파크의 IoT 서비스는 제어, 주차, 출입 등의 기능을 개별앱으로 제공하고 있었습니다.
이 기능을 하나의 앱으로 통합하며, 경쟁사인 푸르지오, 래미안과 차별점을 갖는 UX가 필요했습니다.
마켓 리서치와 임직원 인터뷰, 유저 리서치를 통해 UX콘셉트 및 구체적인 실행 전략을 설정하였습니다.

메인 화면에 날씨 정보를 제공하는 것은 이미 고객사에서 결정된 사항이었습니다.
다만 디자인 시안은 좀처럼 좋은 평가를 받지 못했습니다.

날씨 아이콘을 심플한 것으로 바꾸고 애니메이션 인터랙션을 요청하였습니다.
저는 레퍼런스를 많이 수집하는 편으로, 네이버 와치의 날씨 디자인과 효과를 참고로 하여 전달하였습니다. 디자이너에게 온도 표기 등 크게 적용되는 글자를 light 서체로 바꿔달라고 하였습니다.

다른 작업을 하는 동료에게 피드백을 하는 것이 쉽지 않습니다.
다른 팀원들에게 동의를 구하고, 공감이 되지 않는다면 더 주장하지는 않습니다.
그래서 방향성에 공감하고, 뛰어난 작업을 해내는 동료가 참 중요하다고 생각합니다.

콘텐츠와 템플릿

조건이 성립되면 자동으로 동작하도록 설정하는
자동화 기능은 다른 제어 기능보다 더 어렵습니다.
스토리 방식의 콘텐츠와 그대로 따라서 설정하는
템플릿 기능도 기획하였습니다.

자동화 기능은 2차 프로젝트에 도입되었습니다.

자동화, 실험실 등의 기능이 도입된 것은 고객사의 사정때문입니다. 새로운 기능과 고도화를 꾸준히 진행하고 있음을 어필해야 했기 때문입니다.
1차에서 진행한 제어 기능도 사용자에게 익숙하지 않은데, 자동화는 사용자가 어떻게 받아들일까 생각했습니다.

기능이 결정되었다고 하더라도, 단순히 기능만 제공하지 않는 것이 중요하다고 판단했습니다.
애플 홈이나 네이버페이 등도 콘텐츠 형태의 안내 화면을 제공하고 있었습니다.
보다 잘 이해되기 위해, 누가+언제+무엇을 형태의 콘텐츠를 작성하고, 이 예시를 바탕으로 조금만 바꿔 템플릿을 설정할 수 있도록 하였습니다. 템플릿 기능은 이전 회사의 전자책 프로그램 기획 당시, 사용성을 염두하여 만들었던 템플릿 작업 경험으로 기획할 수 있었습니다.
